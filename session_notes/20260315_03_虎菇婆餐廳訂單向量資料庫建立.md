#### 向量資料庫規格

| 項目 | 設定 |
|------|------|
| **工具** | ChromaDB（本地持久化）|
| **Collection** | `tiger_orders` |
| **位置** | `restaurant-tiger/vector_db_tiger/` |
| **Embedding 模型** | `intfloat/multilingual-e5-small` |
| **向量維度** | 384 維 |
| **距離函數** | Cosine 相似度 |
| **E5 前綴規格** | 文件加 `"passage: "`，查詢加 `"query: "` |

#### 氣象資料對齊邏輯

```
下單時間 → 取整點（floor to hour）→ 查詢氣象字典 → 附加氣溫 / 降雨
```

若當時段無氣象資料，氣溫填入 `-99.0`，降雨填入 `-1.0`（可在 query 時過濾）。

## 二、Embedding 設計說明

每筆訂單的 text 格式：

```
passage: 品項：{品項} 用餐：{用餐型態} 點餐平台：{點餐平台} 交易：{交易型態} 金額：{總金額}元 氣溫：{temp}℃ 降雨：{rain}mm [備註：{備註}]
```

**為什麼用 multilingual-e5-small？**
- 384 維輕量模型，速度快
- 支援繁體中文語意
- E5 系列在通用語意相似度任務上表現優於 MiniLM
- 與美業訪談使用的 `paraphrase-multilingual-MiniLM-L12-v2` 刻意區隔（不同 DB，不同模型）
